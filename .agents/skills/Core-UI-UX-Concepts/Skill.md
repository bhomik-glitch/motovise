# Core UI/UX Concepts Builder

When building or auditing a website UI, ensure the following core concepts are implemented:

*   **Signifiers and Affordances:** The UI should naturally tell the user how it works without instructions [21]. Use button press states, highlights on active navigation items, and hover states to show what a UI element can do [21].
*   **Visual Hierarchy:** Guide the user's eye using size, position, and color [21, 22]. Place the most important elements near the top, make them larger, and use bold colors, while less important information should be smaller and placed below [22]. Use images whenever possible to add a pop of color and aid scanning [22, 23].
*   **Grids and Whitespace:** Use columns as general guidelines for responsive layouts (e.g., 8 columns for tablet, 4 for mobile) [23]. Implement a 4-point grid system where spacing is built in multiples, allowing you to split elements in half while maintaining consistency [24].
*   **Typography:** Stick to a single sans-serif font for the design [24]. Improve headers instantly by tightening letter spacing by -2 to -3% and dropping line height to 110-120% [24, 25].

### Example Usage:
**User Prompt:** "How can I improve the visual hierarchy of my pricing card? Right now, the price, the location, and the 'Buy' button are all the same size and color."
**Skill Output:** "To establish visual hierarchy, use size, position, and color to your advantage [21, 22]. Make the price the largest element, right-aligned, and color it (like blue) to draw the eye [22]. Place the less important location text below it in a smaller font [22]. Ensure the 'Buy' button clearly signifies its affordance with a distinct background color, hover state, and press state [21]."
