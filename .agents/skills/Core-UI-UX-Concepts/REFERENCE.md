# Supplemental Core Concepts Reference Guide

*   **Font Sizing Limits:** Keep the number of font sizes to a maximum of six for standard landing pages [25]. For data-heavy interfaces like dashboards, text generally shouldn't exceed 24 pixels due to the need for high information density [25].
*   **Icon and Button Sizing:** Size your icons perfectly by matching them to the exact line height of your font (e.g., a 24-pixel icon next to text with a 24-pixel line height) [26]. For standalone button padding, a good guideline is to double the button's height to calculate its width [26]. Every button must feature at least four states: default, hovered, active/pressed, and disabled [26, 27].
*   **Micro-interactions:** Build small, purposeful animations to confirm user actions, such as a "copied" chip sliding up when a user clicks a copy button [27].
*   **Image Overlays:** Ensure text remains readable when placed over images. Use a linear gradient that smoothly transitions into a text-readable background, or apply a progressive blur on top of the gradient for a polished look [27, 28].
*   **Shadow Calibration:** Keep in mind that popovers or floating content that sits above other content require stronger shadows than flat cards [12].
